# Flatpak Icons

This directory contains the application icons that are bundled into the
Flatpak build of Aurora Pulse.

## TL;DR — Why doesn't Linux round my icon corners automatically?

Because it can't, by design.

Unlike Android (adaptive icons + system mask) or iOS (rounded-rect mask
applied to every app), the Linux desktop stack does **not** apply a global
shape mask to application icons:

- **GNOME Shell / GNOME Software**: renders the icon as-is.
- **KDE Plasma / Discover**: renders the icon as-is. Some themes apply
  masks to *system* icons but not generally to third-party apps.
- **XFCE / Cinnamon / MATE**: render the icon as-is.
- **Icon themes** (Papirus, Tela, WhiteSur, …): replace third-party app
  icons with their own redrawn versions. The application has no
  influence on this.

If you want a modern rounded look, the rounded shape has to be **baked
into the PNG/SVG** itself. That's what we do here.

## Regenerating

After replacing or updating the master icon in `assets/icons/`, run:

```bash
./flatpak/scripts/generate-icons.sh
```

This will:

1. Render the master into every hicolor PNG size used by GNOME / KDE.
2. Copy the master to `1024x1024/`.
3. Wrap the master in a minimal `scalable/apps/*.svg` so HiDPI displays
   pick the scalable variant first.
4. Use `magick` / `convert` if available (best quality), otherwise fall
   back to macOS `sips`.

The Flatpak manifest (`app.better_craft.aurorapulse.yml`) iterates over
all available sizes during the build, so you don't have to edit it when
adding/removing sizes — just re-run the generator.

## Replacing the icon shape

The current squircle is generated by the design pipeline that lives in
`assets/icons/`. If you want to change the corner radius / shape:

1. Edit the master vector source (Figma / Affinity / Inkscape) and
   re-export `assets/icons/icon-squircle.png` at **1024 × 1024 RGBA**.
2. Make sure the artwork has ~10 % padding to the canvas edge — GNOME
   HIG recommends this so icons don't visually clip in dense grids.
3. Run `./flatpak/scripts/generate-icons.sh`.
4. Commit `flatpak/icons/` and `assets/icons/icon-squircle.png` together.

## Why we ship a "scalable" SVG that wraps a PNG

A real hand-drawn vector SVG would be smaller and sharper at every size,
and is what GNOME Software actually prefers. Until we have that, an SVG
that embeds the high-resolution PNG is still better than no scalable
icon at all: GNOME and KDE prefer `scalable/` over fixed sizes for HiDPI
fallback, so without it their fallback chain ends at 256 px and gets
upscaled badly on 4K screens.

When/if a true vector master becomes available, drop it in directly as
`flatpak/icons/scalable/apps/app.better_craft.aurorapulse.svg` (the
generator will leave a hand-edited SVG alone if you adjust the script
to skip writing it).

## Things Linux cannot fix from the application side

Even with a perfectly rounded squircle PNG you may still see:

- A different icon if the user has installed an icon theme like
  **Papirus** that ships its own version of "Aurora Pulse". This is a
  user choice, not something the app can override.
- A square icon in some legacy launchers / file managers that ignore
  the alpha channel. Affects only very old setups.

Both are out of our control. The squircle PNGs in this directory are
what GNOME Software, vanilla GNOME Shell, vanilla KDE and the Flatpak
software UI will display.
